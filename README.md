## About

I build **local-first AI tools**: persistent agent memory, spec-driven completion loops, and practical local model routing. The through-line is observable systems that survive real use—not dashboards, not vanity metrics.

**Building now:** [cortex](https://github.com/AdityaVG13/cortex) · [gpr](https://github.com/AdityaVG13/gpr) · [Model-Switchboard](https://github.com/AdityaVG13/Model-Switchboard)

## Start Here

Three projects worth your time if you care about agent tooling and local inference.

### [Cortex](https://github.com/AdityaVG13/cortex)

Persistent shared memory and context compression for AI coding agents. Local-first Rust daemon with HTTP, MCP, and a desktop control center.

`Rust` · `MCP` · `local-first` · `agent memory`

### [Model Switchboard](https://github.com/AdityaVG13/Model-Switchboard)

Local model routing for Mac—pick models, switch fast, stay off the cloud when you can.

`Python` · `local models` · `Mac`

### [gpr](https://github.com/AdityaVG13/gpr)

Goal-driven PRD ratchet: an agent loop that only marks work done when real artifacts pass real checks.

`Python` · `spec-driven` · `agent workflows`

## Working Style

- Local-first when the machine can do the job.
- Evidence over vibes: artifacts, tests, and visible completion checks.
- Fast loops for AI tooling, careful loops for published work.
- Agent systems treated like infrastructure—not prompt theater.
